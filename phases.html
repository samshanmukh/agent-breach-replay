Agent Breach Replay Project Phases

A phased plan for building Agent Breach Replay from a local proof of value into an OpenAI-ready security replay platform for tool-using AI agents.

Phase Roadmap

0 Foundation

Product Definition and Architecture

Lock the product thesis, system design, privacy posture, and build prompts so every agent or developer works from the same direction.

Deliverables

  • README product direction
  • architecture page
  • goal page
  • phase roadmap

Exit Criteria

  • clear product promise
  • defined system components
  • defined first release scope

Primary Risk

Scope drift into generic observability instead of agent security replay.

1 v0.1

Local Replay Prototype

Build the first working replay studio with local sample traces. No backend required yet. The goal is to prove the visual security story.

Deliverables

  • Next.js studio shell
  • sample trace fixtures
  • timeline replay
  • flow graph
  • security predicate panel

Exit Criteria

  • unsafe chain is visually obvious
  • guardrailed run can be compared
  • demo works without network setup

Primary Risk

The UI looks polished but does not explain why the action was unsafe.

2 v0.2

Trace Schema, SDK, and Detectors

Define the durable trace contract, build the TypeScript SDK, and run deterministic detectors before using LLMs for explanation.

Deliverables

  • JSON trace schema
  • TypeScript trace SDK
  • metadata-only capture mode
  • redaction helpers
  • first detector rules

Exit Criteria

  • SDK emits replay-ready traces
  • detectors explain finding evidence
  • raw data is not required

Primary Risk

Influence tracking feels magical. Keep it explicit with influencedBy links.

3 v0.3

OpenAI Explanations and Report Export

Use OpenAI models to turn deterministic findings into clear incident narratives, guardrail recommendations, and exportable reports.

Deliverables

  • OpenAI report generator
  • guardrail recommendation prompts
  • report export view
  • trace explanation panel

Exit Criteria

  • LLM explains only triggered findings
  • reports cite trace evidence
  • private data remains redacted

Primary Risk

The model overstates certainty. Keep explanations grounded in event IDs.

4 v0.4

Persistence, Moss Retrieval, and Similar Incidents

Add durable storage with Postgres and semantic retrieval with Moss so teams can search incidents and compare recurring failure patterns.

Deliverables

  • Postgres schema
  • project and run storage
  • Moss incident indexing
  • similar incident search
  • retention controls

Exit Criteria

  • runs persist across sessions
  • similar findings can be retrieved
  • tenant boundaries are explicit

Primary Risk

Trace storage grows quickly. Keep payloads small and structured.

5 v0.5

Adapters and Real Agent Integrations

Connect the replay studio to real agent traces, starting with an OpenAI Agents SDK adapter and generic JSON import.

Deliverables

  • OpenAI trace adapter
  • JSON import/export
  • generic tool wrapper example
  • LangGraph integration draft

Exit Criteria

  • external traces normalize cleanly
  • missing security fields are surfaced
  • examples are reproducible

Primary Risk

Adapters lose important context. Make unknown trust explicit, not silent.

6 v1.0

OpenAI-Ready Release

Harden the product for a serious external review: privacy, security, deployment, SDK packaging, examples, and docs.

Deliverables

  • Vercel deployment
  • npm SDK package
  • PyPI SDK package draft
  • privacy documentation
  • demo script

Exit Criteria

  • clear OpenAI-facing demo
  • no raw-data dependency
  • findings are evidence-backed
  • setup is documented

Primary Risk

Overclaiming prevention. Position as security replay and hardening support.

Cross-Phase Principles

Privacy First

Metadata-only by default. Redact locally. Hash sensitive values. Make capture mode visible.

Rules Before LLMs

Deterministic detectors trigger findings. OpenAI models explain, summarize, and recommend.

Security Replay, Not Generic Logs

Focus on trust, influence, authority, tool boundaries, policy decisions, and evidence.

Recommended Next Step

Start Phase 1 by creating the Next.js replay studio shell, shared trace fixtures, and one end-to-end Vendor Email Assistant scenario. That gives the project a working product spine before deeper platform work.
